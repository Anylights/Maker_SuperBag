# 俯视角射击（Top-Down Shooter）系统策划案（可直接开发）

版本：`v1.0`  
更新时间：`2026-04-09`  
文档定位：`系统设计 + 程序实现规范 + 数值框架 + 开发拆分`

---

## 0. 项目定义

### 0.1 一句话玩法
玩家在俯视角战场中操作角色进行实时射击，从基础武器起步，通过“背包格子中的圣物与石板布局”构筑流派，在一局内完成成长并挑战 Boss。

### 0.2 核心特征
- 俯视角双摇杆射击（移动与瞄准解耦）
- 背包空间管理（圣物占格）
- 石板相邻/范围强化（位置策略）
- Combo 阈值成长（标签联动）
- 轻肉鸽局内升级 + 局外元进度

### 0.3 目标平台
- PC（键鼠）优先
- 后续可扩展手柄（双摇杆逻辑天然兼容）

## 1. 视角与操作系统

## 1.1 相机规则（俯视角）
- 相机投影：透视相机（非正交）
- 相机俯角：`55°`
- 相机距离：`14m`
- 相机跟随：平滑跟随，阻尼 `0.08s`
- 屏幕安全边缘：角色不离开屏幕中心 `35%` 区域
- 禁止镜头旋转，确保空间判断稳定

### 1.2 输入映射（PC）
- 移动：`WASD`
- 瞄准：`鼠标位置`
- 开火：`鼠标左键（长按连射）`
- 技能1：`Q`
- 技能2：`E`
- 换弹：`R`
- 冲刺：`Space`
- 交互：`F`
- 背包：`Tab`
- 地图：`M`

### 1.3 战斗中开背包规则
- 打开背包后进入慢速状态：`TimeScale = 0.15`
- 玩家可拖拽和旋转道具
- 敌人和投射物继续更新（低速）
- 关闭背包后恢复正常速度

## 2.1 局内核心循环

### 2.2 单局时长目标

- 速通局：`16~20 分钟`

### 2.3 章节结构
- 第1章：教学与成型起步（8分钟）
- 第2章：构筑压力与敌人机制（8分钟）

## 4. 背包系统（核心）

## 4.1 背包规格
- 初始格子：`6 x 6`
- 最大扩容：`8 x 8`
- 旋转支持：`90°`
- 非暂停背包：战斗慢速不中断

### 4.2 物品类型
- 圣物（Artifact）：战斗效果主体
- 石板（Tablet）：位置强化器
- 消耗道具（Consumable）：一次性触发，不常驻背包

### 4.3 尺寸定义

| 尺寸 | 典型用途 |
|---|---|
| 1x1 | 小幅数值圣物、工具石板 |
| 1x2 | 常规属性圣物 |
| 2x2 | 核心机制圣物 |
| 2x3 | 大型机制件 |
| 3x3 | 传奇核心件 |

### 4.4 摆放规则
- 圣物必须放入空格才生效
- 石板也占格
- 石板效果按模板作用到相邻/范围格子
- 同格不可叠放
- 背包外圣物不生效

### 4.5 背包收益计算顺序
1. 读取圣物基础属性  
2. 应用圣物等级成长  
3. 应用石板位置加成  
4. 应用 Combo 阈值加成  
5. 应用全局升级和临时 Buff  
6. 进入最终战斗属性

## 6. 石板系统设计（本案关键差异化）

## 6.1 石板定位
石板不是数值件，而是“布局增强器”。  
目标：让玩家围绕核心圣物做空间决策，而非无脑堆词条。

### 6.2 石板作用模板

| 模板名 | 作用范围 | 典型用途 |
|---|---|---|
| 邻接十字 | 上下左右4格 | 强化中心构筑 |
| 九宫格 | 3x3 | 中期泛用强化 |
| 横条 | 同行1xN | 强化弹道排布 |
| 竖条 | 同列Nx1 | 强化召唤排布 |
| 角扇区 | 对角3格 | 特殊布局玩法 |
| 全局弱化+局部强化 | 全图-5%，核心区+20% | 高风险高收益 |

### 6.3 石板正负并存规则
- 50% 石板带负面（如：非范围格 -8% 伤害）
- 负面用于制造布局取舍
- 禁止出现“纯正收益且无代价”的高稀有石板

### 6.4 石板叠加规则
- 同一圣物最多吃到 `3` 个石板增强
- 超出后进入衰减：第4个及以后只生效 `35%`

## 13.4 数学化难度设计（心流导向）

本节给出可直接实现的公式系统，目标是让难度稳定落在“紧张但可控”的心流区间。

### A. 玩家能力值（Skill Score）S

先把战斗表现转为 0~100 分，再做平滑：

`S_raw = 100 * (0.30*Hit + 0.20*Dodge + 0.25*DPSRatio + 0.15*Survive + 0.10*Resource)`

字段定义：
- `Hit`：命中率（0~1）
- `Dodge`：成功规避率（0~1）
- `DPSRatio`：`实际DPS / 章节期望DPS`，再 clamp 到 0~1.5 后归一
- `Survive`：无伤时长得分（0~1）
- `Resource`：金币与补给利用率（0~1）

平滑（EWMA）：

`S_t = (1 - α) * S_(t-1) + α * S_raw`

默认参数：`α = 0.125`（建议范围 `0.08~0.20`）

说明：使用 EWMA 可避免“上一波打得差 -> 下一波骤降难度”的抖动。

### B. 关卡挑战值（Challenge Score）C

按当前房间敌人配置计算挑战值：

`C = Σ_i [ n_i^γ * sqrt(HP_i * DPS_i) * AI_i * Mob_i * Range_i ]`

字段定义：
- `n_i`：第 i 类敌人数
- `γ`：群体协同指数（默认 `1.25`，范围 `1.1~1.4`）
- `HP_i`、`DPS_i`：敌人基础参数
- `AI_i`：AI 威胁系数（1.0~1.4）
- `Mob_i`：机动系数（0.9~1.2）
- `Range_i`：远程压制系数（0.9~1.3）

说明：`n^γ` 用于体现“怪越多越难处理”的非线性压力。

### C. 心流误差（Flow Error）e

定义目标挑战线：

`C_target = k * S_t + b`

`e = C - C_target`

推荐参数：
- `k = 1.05`
- `b = 8`
- 心流带宽 `δ = 6`

判定：
- `e < -δ`：偏简单（接近无聊区）
- `|e| <= δ`：心流区
- `e > +δ`：偏困难（接近焦虑区）

### D. 实时调难（AIMD）

使用加性增加 + 乘性降低，避免难度忽上忽下：

- 若 `e < -δ`（太简单）  
  `C_(t+1) = C_t + a`
- 若 `e > +δ`（太困难）  
  `C_(t+1) = C_t * (1 - β)`
- 若 `|e| <= δ`  
  `C_(t+1) = C_t`

默认参数：
- `a = 2.5`
- `β = 0.12`

这样处理会让变难“慢一点”、变简单“快一点”，能保护心流和挫败感阈值。

### E. 挑战值到实际参数的映射（Logistic 封顶）

将挑战修正量映射到敌人倍率，避免指数炸穿：

`m(x) = m_min + (m_max - m_min) / (1 + exp(-r*(x - x0)))`

用于：
- `EnemyHPMultiplier = m(C_norm)`
- `EnemyDamageMultiplier = m(C_norm_dmg)`
- `SpawnRateMultiplier = m(C_norm_spawn)`

推荐上限：
- HP 倍率：`0.85 ~ 1.35`
- 伤害倍率：`0.85 ~ 1.25`
- 刷怪倍率：`0.90 ~ 1.30`

### F. 胜率目标（Elo 形态概率函数）

用类似 Elo 的逻辑函数监控可通关性：

`P_clear = 1 / (1 + 10^((C* - S*) / D))`

其中：
- `S*`：玩家能力标准分（映射到 Elo 尺度）
- `C*`：关卡难度标准分
- `D`：敏感度（默认 `400`）

目标区间（普通模式）：
- 新手：`P_clear = 0.55~0.70`
- 熟练：`P_clear = 0.70~0.82`

若连续 3 局 `P_clear < 0.45`，自动降低下一局起始挑战 `6%`。

### G. 单局成长曲线（经验）

推荐经验需求曲线：

`XP_to_Level(L) = A * L^p + B`

默认参数：
- `A = 22`
- `p = 1.85`
- `B = 35`

目标：前 5 分钟升级频繁，中后期拉开间隔，避免后期升级噪音。

## 13.6 调参流程（两周内可落地）

### 第1阶段：离线回放（3天）
- 固定 3 套技能画像（新手/普通/高手）
- 跑 300 局模拟，记录 `P_clear`、平均受伤、平均局长
- 先把 `flow_k, flow_b, flow_delta` 调到目标区间

### 第2阶段：小规模实测（4天）
- 20 名玩家，每人至少 5 局
- 重点看“第2章死亡率”和“Boss 到达率”
- 若第2章死亡率 > 45%，优先下调 `aimd_add` 或增大 `flow_delta`

### 第3阶段：线上灰度（7天）
- 对比组：`静态难度` vs `DDA难度`
- 指标：留存、单局时长、连续游玩局数、放弃率
- 目标：DDA 组连续 3 局占比提升 >= 15%

### 14.3 关键结算伪代码

```pseudo
function RebuildPlayerStats():
  stats = BaseStats
  tags = {}

  for artifact in inventory.artifacts:
    stats += artifact.base + artifact.level_growth
    tags.add(artifact.tags)

  for tablet in inventory.tablets:
    affected = tablet.GetAffectedCells(inventory.grid)
    ApplyTabletBonus(stats, affected, tablet)

  activeCombos = ComboModule.Evaluate(tags)
  for combo in activeCombos:
    stats = ApplyCombo(stats, combo)

  stats = ApplyGlobalUpgrades(stats)
  stats = ClampByCaps(stats)
  return stats
```

## 19. 风险与预案

### 风险1：背包系统深度高，但战斗爽感不足
- 预案：前4周优先确保射击/命中反馈成立，再叠系统层

### 风险2：石板复杂导致新手门槛高
- 预案：第1章只投放“纯正向石板”，第2章再引入负面石板

### 风险3：Combo 过多难以读懂
- 预案：MVP 控制 10 条，UI 只重点展示当前主Combo

### 风险4：随机导致构筑断档
- 预案：保留偏向签 + 精英房定向奖励
